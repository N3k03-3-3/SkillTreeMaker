# .agent Directory

このディレクトリは、AI アシスタントと開発者の協働を支援するための**プロジェクト固有のガイドライン**を格納します。

---

## 📁 ディレクトリ構造

```
.agent/
├── skills/                 # 役割別ガードレールルール
│   ├── programmer/
│   │   └── SKILL.md       # プログラマー向けガイドライン
│   ├── code_reviewer/
│   │   └── SKILL.md       # コードレビュアー向けガイドライン
│   └── designer/
│       └── SKILL.md       # デザイナー向けガイドライン
└── workflows/              # 開発ワークフロー定義（将来追加）
```

## 🎯 Skills の目的

**Skills** は、チームメンバーが作業する際に従うべき**ガードレール（安全柵）**を定義します。

### なぜ Skills が必要か

1. **一貫性の確保**: チーム全員が同じルールに従う
2. **品質の保証**: 必須チェックリストで品質を維持
3. **効率の向上**: 何をすべきか明確になり、迷わない
4. **知識の共有**: ベストプラクティスを文書化

---

## 📚 各 Skill の概要

### 💻 Programmer Skill

**対象**: コードを実装する開発者  
**ファイル**: [`skills/programmer/SKILL.md`](skills/programmer/SKILL.md)

**チェックポイント**:
- 実装前: 仕様・設計・規約の確認
- 実装中: コメント・型・エラー処理
- 実装後: セルフレビュー・動作確認

### 👀 Code Reviewer Skill

**対象**: コードレビューを担当する開発者  
**ファイル**: [`skills/code_reviewer/SKILL.md`](skills/code_reviewer/SKILL.md)

**レビューレベル**:
- Level 1 (CRITICAL): 必須項目（違反は差し戻し）
- Level 2 (RECOMMENDED): 推奨項目（改善提案）
- Level 3 (OPTIONAL): ベストプラクティス

### 🎨 Designer Skill

**対象**: UI/UXデザイン・アセット作成担当  
**ファイル**: [`skills/designer/SKILL.md`](skills/designer/SKILL.md)

**品質基準**:
- 命名規則（snake_case 必須）
- ファイル形式（PNG 32bit RGBA）
- 最適化（1MB 以下）

## 🔄 Skills の使い方

### 新規メンバーのオンボーディング

1. **自分の役割の Skill を読む**
   ```bash
   # プログラマーの場合
   cat .agent/skills/programmer/SKILL.md
   ```

2. **チェックリストをブックマーク**
   - 作業前に必ず確認

3. **実践で学ぶ**
   - ルールに従って作業
   - レビューで改善

### AI アシスタントとの協働

このディレクトリは、AI アシスタント（Claude、Cursor など）が自動的に参照します。

```
"プログラマースキルに従ってコードを実装してください"
```

と指示すると、AI が `.agent/skills/programmer/SKILL.md` を読み、ルールに従ったコードを生成します。

## 📝 Skills の更新

Skills は**固定ではなく進化**します。

### 更新プロセス

1. **問題の発見**: ルールに不備や改善点を発見
2. **提案**: Issue やミーティングで提案
3. **議論**: チームで合意形成
4. **更新**: Skill ファイルを更新

### 更新時の注意

- **後方互換性**: 既存コードが壊れないように
- **明確化**: ルールは具体的で明確に
- **例示**: 良い例・悪い例を必ず含める

## ❓ FAQ

### Q: Skills とコーディング規約の違いは？

**A**: 
- **コーディング規約**: コードの書き方の基準（技術的）
- **Skills**: 作業プロセス全体のガイド（プロセス的）

Skills はコーディング規約を包含し、さらに作業フロー、チェックリスト、ベストプラクティスを含みます。

### Q: 新しい Skill を追加できますか？

**A**: はい！新しい役割（例: テスター、テクニカルライター）が追加された場合、対応する Skill を作成してください。

```
.agent/skills/
└── tester/
    └── SKILL.md
```

### Q: Skill に従わないとどうなりますか？

**A**: 
- **必須ルール違反**: コードレビューで差し戻し
- **推奨ルール違反**: 改善提案
- **チーム全体の品質低下**: 技術的負債の蓄積

---

**Skills は、チーム全体の生産性と品質を支える重要な資産です。積極的に活用・改善してください。**
